# 代码审查清单

## 1. 功能性
- [ ] 代码是否实现了需求
- [ ] 边界条件是否处理
- [ ] 错误处理是否完善

## 2. 代码质量
- [ ] 命名清晰易懂
- [ ] 函数不要太长（<50行）
- [ ] 没有重复代码
- [ ] 适当的注释

## 3. 安全性
- [ ] 没有硬编码密码/密钥
- [ ] 输入验证
- [ ] SQL 注入防护
- [ ] XSS 防护

## 4. 性能
- [ ] 没有明显的性能问题
- [ ] 数据库查询优化

## 5. 测试
- [ ] 有测试用例
- [ ] 测试通过
